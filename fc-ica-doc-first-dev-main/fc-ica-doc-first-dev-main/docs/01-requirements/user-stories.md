# 사용자 스토리 (User Stories)

## 프로젝트: TaskFlow — 태스크 관리 SaaS

---

## 1. 인증 (Authentication)

### US-01: 회원가입

> **As a** 신규 사용자
> **I want to** 이메일과 비밀번호로 회원가입을 하고 싶다
> **So that** TaskFlow 서비스를 이용할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 이메일, 비밀번호, 이름을 입력하여 회원가입할 수 있다
- [ ] 이메일 형식이 올바르지 않으면 에러 메시지를 표시한다
- [ ] 비밀번호는 최소 8자 이상이어야 한다
- [ ] 이미 가입된 이메일이면 "이미 사용 중인 이메일입니다" 메시지를 표시한다
- [ ] 회원가입 성공 시 자동으로 로그인되어 대시보드로 이동한다

**우선순위:** P0 (Must)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-01-01

---

### US-02: 로그인

> **As a** 가입된 사용자
> **I want to** 이메일과 비밀번호로 로그인하고 싶다
> **So that** 내 프로젝트와 태스크에 접근할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 이메일과 비밀번호를 입력하여 로그인할 수 있다
- [ ] 로그인 성공 시 JWT 토큰이 발급되고 대시보드로 이동한다
- [ ] 이메일 또는 비밀번호가 틀리면 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지를 표시한다
- [ ] 로그인 상태는 브라우저를 닫았다 다시 열어도 유지된다 (Refresh Token)
- [ ] 로그인하지 않은 사용자는 로그인 페이지로 리다이렉트된다

**우선순위:** P0 (Must)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-01-02

---

### US-03: 로그아웃

> **As a** 로그인된 사용자
> **I want to** 로그아웃하고 싶다
> **So that** 다른 사람이 내 계정에 접근하지 못하도록 할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 헤더의 프로필 메뉴에서 "로그아웃" 버튼을 클릭할 수 있다
- [ ] 로그아웃 시 클라이언트의 토큰이 삭제된다
- [ ] 로그아웃 후 로그인 페이지로 이동한다
- [ ] 로그아웃 후 보호된 페이지에 접근하면 로그인 페이지로 리다이렉트된다

**우선순위:** P0 (Must)
**예상 작업량:** 1 Story Point
**연관 기능:** FR-01-03

---

## 2. 프로젝트 관리 (Project Management)

### US-04: 프로젝트 생성

> **As a** 팀 리더
> **I want to** 새로운 프로젝트를 생성하고 싶다
> **So that** 팀의 업무를 프로젝트 단위로 구분하여 관리할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 대시보드에서 "새 프로젝트" 버튼을 클릭하면 생성 모달이 열린다
- [ ] 프로젝트명(필수)과 설명(선택)을 입력할 수 있다
- [ ] 프로젝트명은 1~100자 이내여야 한다
- [ ] 생성 완료 시 해당 프로젝트의 보드 화면으로 이동한다
- [ ] 생성한 사용자는 자동으로 프로젝트 소유자(owner)가 된다

**우선순위:** P0 (Must)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-02-01

---

### US-05: 프로젝트 목록 조회

> **As a** 사용자
> **I want to** 내가 속한 프로젝트 목록을 보고 싶다
> **So that** 원하는 프로젝트를 빠르게 선택하여 들어갈 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 대시보드에 내가 속한 프로젝트가 카드 형태로 표시된다
- [ ] 각 카드에 프로젝트명, 설명, 멤버 수, 태스크 요약(상태별 개수)이 표시된다
- [ ] 프로젝트 카드를 클릭하면 해당 프로젝트의 보드 화면으로 이동한다
- [ ] 프로젝트가 없는 경우 "프로젝트를 생성해 보세요" 안내 메시지를 표시한다

**우선순위:** P0 (Must)
**예상 작업량:** 2 Story Points
**연관 기능:** FR-02-02

---

### US-06: 멤버 초대

> **As a** 프로젝트 소유자
> **I want to** 팀원을 프로젝트에 초대하고 싶다
> **So that** 팀원들이 함께 태스크를 관리할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 프로젝트 설정에서 "멤버 초대" 버튼을 클릭할 수 있다
- [ ] 초대할 사용자의 이메일을 입력하여 초대할 수 있다
- [ ] 가입되지 않은 이메일이면 "가입된 사용자가 아닙니다" 메시지를 표시한다
- [ ] 이미 프로젝트 멤버인 경우 "이미 멤버입니다" 메시지를 표시한다
- [ ] 초대된 사용자의 대시보드에 해당 프로젝트가 즉시 표시된다
- [ ] 프로젝트 소유자만 멤버를 초대할 수 있다

**우선순위:** P1 (Should)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-02-06

## 3. 태스크 관리 (Task Management)

### US-07: 태스크 생성

> **As a** 프로젝트 멤버
> **I want to** 새로운 태스크를 생성하고 싶다
> **So that** 해야 할 업무를 기록하고 추적할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 보드뷰 상단 또는 컬럼 하단의 "+" 버튼으로 태스크를 생성할 수 있다
- [ ] 제목(필수), 설명(선택), 우선순위(선택, 기본값: MEDIUM), 담당자(선택)를 입력할 수 있다
- [ ] 새로 생성된 태스크의 기본 상태는 "TODO"이다
- [ ] 생성 완료 시 해당 태스크가 TODO 컬럼에 즉시 표시된다
- [ ] 제목은 1~200자 이내여야 한다

**우선순위:** P0 (Must)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-03-01

---

### US-08: 태스크 수정

> **As a** 프로젝트 멤버
> **I want to** 태스크의 내용을 수정하고 싶다
> **So that** 변경된 요구사항이나 세부 정보를 반영할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 태스크 카드를 클릭하면 사이드 패널에 상세 정보가 표시된다
- [ ] 제목, 설명, 우선순위, 담당자, 상태를 수정할 수 있다
- [ ] 수정 사항은 저장 버튼 클릭 시 반영된다
- [ ] 수정 성공 시 보드뷰에 즉시 반영된다

**우선순위:** P0 (Must)
**예상 작업량:** 2 Story Points
**연관 기능:** FR-03-04

---

### US-09: 태스크 삭제

> **As a** 프로젝트 멤버
> **I want to** 불필요한 태스크를 삭제하고 싶다
> **So that** 보드를 깔끔하게 유지할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 태스크 상세 패널에서 "삭제" 버튼을 클릭할 수 있다
- [ ] 삭제 전 "정말 삭제하시겠습니까?" 확인 다이얼로그가 표시된다
- [ ] 삭제 확인 시 태스크가 보드에서 제거된다 (소프트 삭제)
- [ ] 삭제된 태스크는 목록에 더 이상 표시되지 않는다

**우선순위:** P0 (Must)
**예상 작업량:** 1 Story Point
**연관 기능:** FR-03-05

---

### US-10: 태스크 상태 변경

> **As a** 프로젝트 멤버
> **I want to** 태스크의 상태를 변경하고 싶다
> **So that** 업무 진행 상황을 정확히 반영할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 태스크 상세 패널에서 상태 드롭다운으로 변경할 수 있다
- [ ] 상태 값은 TODO, IN_PROGRESS, DONE 중 하나이다
- [ ] 상태 변경 시 보드뷰의 해당 컬럼으로 카드가 이동한다
- [ ] 상태 변경은 즉시 저장된다 (별도 저장 버튼 불필요)

**우선순위:** P0 (Must)
**예상 작업량:** 2 Story Points
**연관 기능:** FR-03-06

---

### US-11: 태스크 필터링

> **As a** 프로젝트 멤버
> **I want to** 태스크를 담당자, 우선순위, 상태로 필터링하고 싶다
> **So that** 원하는 태스크만 빠르게 찾을 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 보드뷰 상단에 필터 드롭다운이 표시된다
- [ ] 담당자, 우선순위(LOW/MEDIUM/HIGH/URGENT)로 필터링할 수 있다
- [ ] 필터 적용 시 해당 조건에 맞는 태스크만 보드에 표시된다
- [ ] 필터 초기화 버튼으로 모든 필터를 해제할 수 있다
- [ ] 복수 필터를 동시에 적용할 수 있다 (AND 조건)

**우선순위:** P1 (Should)
**예상 작업량:** 3 Story Points
**연관 기능:** FR-03-02

---

## 4. 칸반 보드뷰 (Kanban Board View)

### US-12: 칸반 보드 드래그앤드롭

> **As a** 프로젝트 멤버
> **I want to** 태스크 카드를 드래그하여 다른 컬럼으로 이동하고 싶다
> **So that** 마우스 조작만으로 빠르게 태스크 상태를 변경할 수 있다

**인수 조건 (Acceptance Criteria):**

- [ ] 태스크 카드를 마우스로 드래그할 수 있다
- [ ] 드래그 중 카드가 반투명하게 표시되고, 드롭 가능 영역이 하이라이트된다
- [ ] 다른 컬럼에 드롭하면 태스크 상태가 해당 컬럼의 상태로 변경된다
- [ ] 같은 컬럼 내에서도 순서를 변경할 수 있다
- [ ] 드래그앤드롭 후 API 호출이 실패하면 원래 위치로 롤백되고 에러 메시지를 표시한다
- [ ] 모바일에서는 터치 드래그를 지원한다

**우선순위:** P0 (Must)
**예상 작업량:** 5 Story Points
**연관 기능:** FR-04-02
